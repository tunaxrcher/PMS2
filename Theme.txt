# Serene PMS — Luxury Ambient Glassmorphism Hotel / Resort PMS UI

This file defines the complete visual style for Serene PMS.

Requirement.txt defines what the system must do.
Style.txt defines how the UI must look, feel, and behave visually.

All developers and AI coding tools must read and follow this file before creating or editing any UI.

---

# 1. Main Visual Target

The UI must look like a premium luxury resort / hotel PMS dashboard with warm ambient glassmorphism.

The visual direction is:

* Luxury Ambient Glassmorphism
* Warm Cinematic Glass UI
* Frosted Glass Dashboard
* Premium Hotel PMS Dashboard
* Luxury Resort Control Center
* Operational SaaS Dashboard with premium glass styling
* Adobe-style app navigation with better PMS usability

The UI should feel:

* Premium
* Warm
* Elegant
* Calm
* Spacious
* Professional
* Modern
* Resort-like
* Easy to scan
* Suitable for daily hotel / resort operations

Final visual target:

```txt
A usable luxury hotel PMS dashboard with a warm blurred resort background, Adobe-style expanded glass sidebar, glass topbar, and readable frosted-glass panels.
```

This is a business application, not a decorative landing page.
Readability and operational usability are more important than visual effects.

---

# 2. What This UI Must Not Look Like

Do not make the UI look like:

* Generic dark admin dashboard
* Generic light admin dashboard
* Old Bootstrap admin template
* Cheap ERP software
* Plain SaaS template
* Flat navy dashboard
* Flat black dashboard
* Neon cyberpunk interface
* Gaming RGB interface
* Overly colorful landing page
* Decorative concept UI that is hard to use
* Boring table-only back office system
* A stack of large pill buttons in the sidebar
* Icon-only sidebar on desktop by default

Important:

If the result only looks like a normal dark dashboard, it is not good enough.

The goal is not just “dark mode”.
The goal is:

```txt
Luxury glass PMS dashboard + strong operational UX
```

---

# 3. UX Direction

Serene PMS is an operational system used by hotel / resort staff every day.

Therefore:

* Navigation must be easy to understand
* Desktop menu must show both icons and labels
* Sidebar must be readable for new staff
* Dense pages must remain readable
* Room Grid, Booking, Payments, and Reports need wide usable space
* Glass effects must not hurt readability
* Background must support the UI, not dominate it

Preferred UX direction:

```txt
Adobe-style expanded sidebar usability
+
Luxury resort glass visual mood
+
Readable PMS operation layout
```

Do not use an icon-only sidebar as the default desktop layout.

Icon-only sidebar is allowed only for:

* Collapsed mode
* Mobile / tablet compact mode
* Optional user preference

Default desktop sidebar must be expanded with icon + label.

# 5. Most Important Glass Rule

Glassmorphism only works if there is something behind the glass.

The app must have:

* A real background image or strong visual background layer
* Warm ambient glow
* Depth behind the panels
* Translucent panels
* Backdrop blur
* Thin highlighted borders

Do not use a flat black or flat navy background.

If there is no background scene, the glass effect will not look real.

The app must use a full-screen luxury background.

Preferred background:

* Luxury resort lounge
* Hotel lobby
* Warm interior
* Spa / villa / lounge mood
* Night resort ambience
* Soft lamps
* Warm bokeh
* Elegant interior lighting

Recommended file path:

```txt

```

The background must be:

* Full screen
* Cover
* Centered
* Slightly blurred or softened
* Darkened with overlay
* Enhanced with warm glow
* Enhanced with vignette

The background must be visible subtly behind the glass panels.

Do not:

* Use flat black only
* Use flat navy only
* Use plain gradient only if an image is available
* Use sharp busy photo behind text
* Use a noisy image that hurts readability
* Let the background overpower the PMS data

Background must support the interface, not compete with it.

# 7. App Background Component

Create a reusable component:

```txt
components/layout/app-background.tsx
```

The AppBackground component must provide:

* Full-screen luxury image background
* Dark overlay
* Warm amber/orange glow
* Optional subtle cyan/blue secondary glow
* Vignette
* Content layer above background

Recommended implementation direction:

```tsx
export function AppBackground({ children }: { children: React.ReactNode }) {
  return (
    <div className="relative min-h-screen overflow-hidden bg-[#120d09] text-stone-100">
      {/* Luxury background image */}
      <div
        className="absolute inset-0 scale-105 bg-cover bg-center opacity-65 blur-[1px]"
        style={{
          backgroundImage: "url('/images/resort-lounge-bg.jpg')",
        }}
      />

      {/* Dark overlay for readability */}
      <div className="absolute inset-0 bg-black/60" />

      {/* Warm ambient glow */}
      <div className="pointer-events-none absolute left-1/2 top-10 h-[420px] w-[420px] -translate-x-1/2 rounded-full bg-amber-300/18 blur-3xl" />
      <div className="pointer-events-none absolute bottom-10 right-10 h-[360px] w-[360px] rounded-full bg-orange-500/18 blur-3xl" />
      <div className="pointer-events-none absolute bottom-0 left-10 h-[300px] w-[300px] rounded-full bg-cyan-400/8 blur-3xl" />

      {/* Vignette */}
      <div className="absolute inset-0 bg-[radial-gradient(circle_at_center,transparent_0%,rgba(0,0,0,0.45)_72%,rgba(0,0,0,0.85)_100%)]" />

      {/* Content */}
      <div className="relative z-10">{children}</div>
    </div>
  )
}
```

If the background image does not exist yet, create the component anyway and leave the image path ready.

# 14. Glassmorphism Rules

All major surfaces should feel like frosted glass.

A proper glass panel must have:

* Transparency
* Backdrop blur
* Thin bright border
* Soft shadow
* Depth from background
* Readable text

Main glass panel style:

```txt
rounded-[28px]
border border-white/15
bg-black/30
backdrop-blur-2xl
shadow-[0_24px_80px_rgba(0,0,0,0.45)]
```

Normal glass card style:

```txt
rounded-2xl
border border-white/15
bg-white/[0.08]
backdrop-blur-xl
shadow-[0_18px_50px_rgba(0,0,0,0.30)]
```

Subtle inner panel:

```txt
rounded-2xl
border border-white/10
bg-black/20
backdrop-blur-xl
```

For dense operational panels, use slightly more opacity:

```txt
bg-black/35
```

Do not:

* Make panels fully opaque
* Use solid dark cards everywhere
* Use blur without a background behind it
* Make text low contrast
* Overuse blur on every small element
* Put table text on noisy transparent areas

Important rule:

```txt
Glass is for major surfaces.
Dense data must remain readable.
```

# 33. Responsive Design

Desktop first.
Tablet second.
Mobile mainly for Housekeeping.

Rules:

* Front Desk screens should work best on desktop/tablet
* Housekeeping screens should work well on mobile
* Tables can become cards on small screens
* Sidebar can collapse on smaller screens
* Room Grid may require horizontal scroll
* Glass effects should be reduced on low-performance devices if needed

---

# 34. Performance Rules

Glassmorphism can be expensive.

Performance rules:

* Use backdrop blur on major panels only
* Do not apply blur to hundreds of table rows
* Do not apply blur to every Room Grid cell
* Keep background effects mostly static
* Avoid excessive animations
* Use virtualization for large Room Grid and tables

If performance drops:

* Increase panel opacity
* Reduce blur intensity
* Reduce number of glow layers
* Reduce animated effects

---

# 35. Accessibility and Readability

Readability is mandatory.

Rules:

* Text contrast must be high
* Important numbers must be easy to read
* Small text must not sit on noisy background
* Focus states must be visible
* Buttons must be large enough
* Status must not rely on color alone
* Use icons and labels when useful
* Sidebar desktop menu must show labels by default

Do not sacrifice usability for decoration.

---

# 36. Things to Avoid

Do not:

* Use flat dark background
* Use flat navy SaaS theme
* Use generic admin dashboard layout
* Use icon-only sidebar as default desktop layout
* Use solid opaque panels everywhere
* Use neon cyberpunk colors
* Use low contrast text
